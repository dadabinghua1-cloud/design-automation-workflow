## 未实现功能

- 上传按钮只是入口占位，不是真正上传功能。
- 当前 UI 是静态查看页面，不会自动运行脚本。
- 当前 UI 不会自动生成图片。
- 当前 UI 不会自动调用 Codex。
- 当前 UI 不会写回 `image_generation_queue.json`。
- 当前 UI 不会写回 `output_register.csv/json`。
- 当前 UI 不接 OpenAI Images API。
- 当前 UI 不接 Figma / Photoshop。

## 当前 UI 定位

当前 UI 定位为 Design Automation Workflow 1.1 的静态查看型管理台。

它适合用于：

- 查看当前项目状态。
- 查看输入素材列表。
- 查看提示词列表和提示词正文。
- 查看已保存的视觉预览图。
- 查看输出登记状态。
- 查看小批量出图队列。

它不负责执行自动化任务。数据刷新仍通过 `scripts/deploy_ui.ps1` 完成。

## 下一步建议

- 保持当前 UI 作为 1.1 查看型管理台。
- 先继续验证真实项目流程，不急于美化 UI。
- v1.2 可增加上传前置流程或本地数据刷新按钮。
- v1.2 / v1.3 再考虑自动运行脚本、API 自动出图或写回队列状态。
- Figma / Photoshop 接入继续放到 v2.0。

## 验收结论

通过 1.1 静态验收。

当前 GitHub Pages 静态 UI 已可用于查看项目状态、素材、提示词、视觉预览、输出登记和项目队列。

`v1.2-ui-usability-draft` 后，UI 更适合作为设计自动化项目管理台查看当前阶段、下一步动作和能力边界。
