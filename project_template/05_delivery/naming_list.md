# 统一命名规范

用途：统一预览图、最终图和可继续处理文件的命名方式，方便客户确认、内部追踪和归档。

## 1. 标准格式

```text
编号_物料名称_尺寸_版本号.格式
```

## 2. 命名示例

```text
01_电子水牌底图_1080x1920_v01.png
02_签到背景板底图_3840x2160_v01.png
03_二维码立牌底图_1080x1920_v01.png
```

## 3. 字段说明

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| 编号 | 与物料清单 id 保持一致 | 01 |
| 物料名称 | 使用清晰中文名称 | 电子水牌底图 |
| 尺寸 | 宽 x 高，中间使用小写 x | 1080x1920 |
| 版本号 | 使用 v 加两位数字 | v01 |
| 格式 | 根据交付要求选择 | png |

## 4. 命名规则

- 文件名不使用空格。
- 不使用特殊符号。
- 不随意改动物料编号。
- 修改版本时递增版本号，例如 `v01`、`v02`、`v03`。
- 同一轮预览保持版本号一致。
- 最终交付文件应与客户确认版本一致。

## 5. 推荐文件位置

| 文件类型 | 存放位置 |
| --- | --- |
| 预览图 | `04_outputs/preview/` |
| 最终图 | `04_outputs/final/` |
| 可继续处理文件 | `04_outputs/ps_ready/` |
| 对外交付文件 | `05_delivery/` |
| 历史版本 | `99_archive/` |
