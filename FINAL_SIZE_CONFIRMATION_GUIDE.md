# 正式尺寸确认流程指南

Design Automation Workflow 中必须区分“视觉预览图”和“最终交付图”。

## 为什么要区分

视觉预览图用于判断风格、构图、留白、信息区位置和整体氛围是否可接受。它不一定等于最终生产尺寸，也不一定满足印刷或投放平台的像素、DPI、出血和安全区要求。

最终交付图用于真实投放、印刷或交付客户。它必须经过尺寸确认、格式确认和交付检查，不能只因为视觉效果通过就直接标记为完成。

## 正式交付前需要确认

正式交付前，需要逐张确认：

- 目标尺寸是什么。
- 当前像素是多少。
- 当前像素是否等于目标像素。
- 是否需要重制。
- 是否需要放大。
- 是否需要进入 PS / Figma 处理。
- 如果是印刷尺寸，是否需要确认 DPI、出血、安全区和实际交付规格。

## 判断原则

- 当前像素等于目标像素：可以进入最终交付确认。
- 当前像素小于目标像素：需要重制、放大或进入 PS / Figma 处理。
- 目标尺寸为米、厘米、A4、A5 等印刷规格：需要后续确认 DPI、出血和实际交付规格。
- 当前阶段只给出建议，不自动判断为最终完成。

## 禁止事项

- 不允许直接把视觉预览图标记为最终交付完成。
- 不允许跳过正式尺寸确认。
- 不允许在未确认 DPI / 出血 / 安全区的情况下把印刷物料标记为最终交付完成。
- 不允许因为风格通过就省略最终尺寸检查。

## 推荐流程

1. 运行 `scripts/check_final_size.ps1`。
2. 查看 `04_outputs/final_size_check_report.md`。
3. 人工确认每张图的 `size_decision`。
4. 决定 `final_action`：重制、放大、进入 PS、进入 Figma、或进入最终交付确认。
5. 更新 `final_size_confirmation.csv/json`。
6. 再进入最终交付检查。
