# 单物料提示词生成指南

本指南说明如何使用 `scripts/generate_prompts.ps1`，根据真实项目中的物料清单和统一提示词模板，生成每个物料对应的提示词文件。

当前脚本只生成 Markdown 文本文件。不接 Figma，不接 Photoshop，不安装依赖，不做图片生成，不写复杂插件。

## 为什么要根据物料清单生成提示词

设计延展项目通常会包含多个尺寸和用途不同的物料。如果手动逐个写提示词，很容易出现：

- 图1、图2、图3顺序混乱
- 尺寸写错
- 横版 / 竖版写错
- 预留区域漏写
- 需要保留或去掉的元素不一致
- 主风格和负面提示词没有统一引用

根据 `material_list.json` 自动生成单物料提示词，可以让每个提示词都来自同一份物料清单，减少重复输入和低级错误。

## 什么时候运行 generate_prompts.ps1

建议在以下条件满足后运行：

1. 已创建真实项目。
2. 已完成项目结构检查。
3. 已填写或更新 `01_review/requirement_check.md`。
4. 已完成第一遍确认和第二遍确认。
5. 已更新 `02_material_list/material_list.json`。
6. 已确认 `03_prompts/master_style_prompt.md` 和 `03_prompts/negative_prompt.md` 可用。

不要在物料清单未确认前运行，否则生成的提示词也会跟着不准确。

## 如何运行 generate_prompts.ps1

在项目根目录打开 PowerShell，运行：

```powershell
.\scripts\generate_prompts.ps1
```

然后输入真实项目名称，例如：

```text
20260611_威元活动物料延展
```

也可以直接带项目名运行：

```powershell
.\scripts\generate_prompts.ps1 -InputProjectName "20260611_威元活动物料延展"
```

脚本默认读取：

```text
projects/20260611_威元活动物料延展/
```

如果 PowerShell 阻止脚本运行，可以在当前窗口临时允许本次运行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\scripts\generate_prompts.ps1
```

## 生成的提示词在哪里

单物料提示词会生成到：

```text
projects/项目名称/03_prompts/per_material_prompts/
```

命名格式：

```text
编号_物料名称_尺寸_prompt.md
```

示例：

```text
01_电子水牌底图_1080x1920_prompt.md
02_签到背景板底图_3840x2160_prompt.md
03_二维码立牌底图_1080x1920_prompt.md
```

生成报告会保存到：

```text
projects/项目名称/03_prompts/prompt_generation_report.md
```

## 生成后如何人工检查

生成后请逐个打开单物料提示词，重点检查：

- 图1、图2、图3顺序是否正确
- 物料名称是否正确
- 尺寸是否正确
- 比例是否正确
- 横版 / 竖版是否正确
- 输出类型是否正确
- 预留区域是否清楚
- 需要保留的元素是否完整
- 需要去掉或不生成的元素是否完整
- 推荐输出文件名是否符合命名规范
- 是否明确写出不生成文字、二维码、Logo、箭头、地址
- 是否明确写出文字、二维码、Logo、箭头、地址均后期添加
- 是否明确写出只做底图

特殊物料还需要额外检查专属规则：

- 签到背景板：是否写明只预留签到主题区，不生成签到字。
- 云摄影二维码立牌：是否写明只预留二维码区，不生成二维码。
- 指引 KT 板：是否写明只预留文字区和箭头区，不生成文字和箭头。
- 倒计时牌：是否写明只预留倒计时数字和标题区，不生成倒计时文字和数字。
- 桌号牌：是否写明预留桌号数字区、保留圆形裁切安全区、不生成桌号数字。
- 参会证：是否写明预留姓名、单位、二维码或编号区，不生成姓名、单位、二维码和编号。
- 餐券：是否写明预留餐券信息、编号或二维码区，不生成餐券文字、编号和二维码。

如果发现问题，先修改 `02_material_list/material_list.json`，再重新运行脚本。

## 为什么提示词生成后仍然需要人工确认

脚本只能根据已有字段拼接提示词，不能判断客户真实意图是否变化，也不能判断主 KV 的视觉重点是否需要特殊处理。

人工确认仍然必要，因为需要确认：

- 提示词是否符合客户最终意见
- 预留区是否满足后期排版
- 是否仍然默认只做底图
- 是否遗漏 Logo、二维码、箭头、地址等后期元素
- 是否需要针对某个物料单独调整表达

提示词生成完成后，应先人工检查，再进入底图延展。

只有提示词通过人工审核后，才可以进入底图延展。提示词未审核前，不要直接生成图片。
