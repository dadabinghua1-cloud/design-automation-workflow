# 小批量输出审核指南

本指南说明 Design Automation Workflow 1.1 中，预览图保存后如何进行本地输出审核。

当前阶段不调用图片生成 API，不接 Figma，不接 Photoshop。

## 为什么要区分视觉预览图和最终交付图

视觉预览图用于判断风格、构图、留白、预留区和整体方向是否可接受。它适合做小批量测试记录，但不一定满足最终交付的精确尺寸、分辨率、印刷出血或安全区要求。

最终交付图用于正式交付客户或进入制作流程，必须确认尺寸、格式、清晰度、命名和版本都符合交付标准。

## 为什么 ChatGPT 生成图可能不是最终精确尺寸

ChatGPT 内部生成的图片通常适合视觉验证，但可能存在：

- 实际像素与目标尺寸不完全一致
- 输出比例接近但不是生产规格
- 不包含印刷出血或安全线
- 需要后续在 PS / Figma 中重制、裁切或整理

因此，视觉通过不等于最终交付完成。

## 如何运行 review_outputs.ps1

在项目根目录打开 PowerShell，运行：

```powershell
.\scripts\review_outputs.ps1 -InputProjectName "20260611_真实小项目测试"
```

也可以不带参数运行：

```powershell
.\scripts\review_outputs.ps1
```

然后按提示输入项目名称。

## 如何查看 output_review_report.md

脚本运行后会生成：

```text
projects/项目名称/04_outputs/output_review_report.md
```

报告会记录：

- 审核时间
- 项目名称
- 审核对象
- 文件是否存在
- 文件名是否规范
- 目标尺寸
- 实际像素尺寸
- 是否为最终生产尺寸
- 视觉测试结论
- 当前审核结论
- 下一步建议

## 什么情况下可以标记为“视觉预览通过”

满足以下条件时，可以标记为：

```text
视觉预览通过，待正式尺寸确认
```

条件：

- 文件已保存到 `04_outputs/preview/`
- 文件名符合队列和命名规范
- 视觉方向已通过小批量测试
- 预留区、构图、风格没有明显问题
- 已明确该文件仍不是最终交付图

## 什么情况下才能标记为“最终交付完成”

必须满足以下条件后，才可以标记最终交付完成：

- 尺寸与最终生产要求一致
- 清晰度和格式符合交付要求
- 文字、二维码、Logo、箭头、地址等后期内容已正确添加
- 文件已通过最终交付检查
- 文件已保存到 `04_outputs/final/`
- 客户或负责人确认通过

## 后续接 API 或 PS / Figma 时如何处理最终尺寸

如果后续接 OpenAI Images API，可以让 API 自动生成接近目标规格的图片，并保存到 `04_outputs/preview/`。但正式交付前仍需要确认实际像素和最终尺寸。

如果后续进入 PS / Figma，应使用视觉预览图作为方向参考，根据最终尺寸重新整理画布、预留区、后期文字和交付版本。
