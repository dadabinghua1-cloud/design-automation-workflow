# 当前版本使用说明

当前版本是 Design Automation Workflow v1.2 可使用草稿版，适合设计师用来管理设计延展项目的状态、素材、提示词、预览图、输出登记和下一步动作。

## 当前版本能做什么

- 查看项目状态。
- 查看素材、KV、PPT 和参考图列表。
- 查看单物料提示词。
- 查看视觉预览图。
- 查看输出登记和出图队列。
- 记录正式尺寸确认和执行准备状态。
- 通过 Next Action Panel 复制下一步给 Codex 的指令。

## 当前版本不能做什么

- 不能网页上传素材。
- 不能自动生成图片。
- 不能自动调用 Codex。
- 不能接 OpenAI Images API 或其他绘画模型 API。
- 不能自动接 Figma / Photoshop。
- 不能替代设计师判断视觉效果。

## 新项目怎么开始

1. 用脚本复制 `project_template` 创建新项目。
2. 把主 KV、PPT、参考图、Logo 和可选尺寸表放入对应文件夹。
3. 让 Codex 先做需求检查和缺失信息整理。
4. 用户完成两遍确认后，再生成提示词。

## 什么时候需要手动 commit / push

当 Codex 更新了流程文件、提示词、报告、状态文件或 UI 数据后，需要手动 commit / push 到 GitHub，GitHub Pages 才能展示最新内容。

## 什么时候需要问 ChatGPT

- 需要判断视觉方向是否好看。
- 需要讨论风格、构图、留白和审美。
- 需要生成或评估视觉预览图。

## 什么时候需要找 Codex

- 需要整理文件夹、清单、报告和状态。
- 需要生成提示词。
- 需要更新 JSON / CSV。
- 需要刷新 GitHub Pages 静态数据。
- 需要生成下一步执行指令。

## 什么时候需要自己判断设计效果

视觉预览是否好看、是否符合客户气质、是否可以继续深化，仍需要设计师人工判断。

## 推荐日常流程

1. 放入素材。
2. 让 Codex 做需求检查。
3. 两遍确认物料和尺寸。
4. 生成提示词。
5. 小批量测试视觉预览。
6. 保存预览图。
7. 跑输出检查和尺寸确认。
8. 打开 GitHub Pages 查看状态。
9. 复制 Next Action 指令继续推进。
